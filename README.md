# SourceGenerators
This solution demonstrates multiple Roslyn source generator approaches for generating CSV serialization helpers at compile time. This is demo code only and has not been tested against production workloads and complex object structures. 

It includes:
- A generator library (`Generators`) with legacy and incremental implementations.
- A sample consumer app (`ConsoleApp1`) that uses generated CSV APIs.
- A test project (`GeneratorTests`) with snapshot tests (Verify) and integration tests.

## What this solution does

The generators scan for classes/records marked with specific attributes and generate CSV methods, including:
- `ToCsv()` for a single instance
- `ToCsvHeader()` for the CSV header row
- `ToCsv(IEnumerable<T>)` for a sequence with header + rows

For string properties, generated code wraps values in quotes.

## Solution layout

- `Generators/`
	- `CsvGenerator.cs`: legacy `ISourceGenerator` implementation (reference-only / older pattern)
	- `CsvIncrementalGenerator.cs`: current primary `IIncrementalGenerator` implementation (recommended)
	- `CsvIncrementalGenerator6.cs`: .NET 6-style incremental implementation for comparison
	- `Helpers/`: helper models and utility types for generation
- `ConsoleApp1/`
	- Sample app consuming generated members from `CsvIncrementalGenerator`
- `GeneratorTests/`
	- Snapshot tests for generated source output
	- Integration tests for runtime behavior of generated CSV methods

## Generator differences

### 1) `CsvGenerator` (legacy)
- Pattern: `ISourceGenerator` + syntax receiver
- Attribute: `CsvSerializable`
- Output shape: generates static extension methods on a generated helper type
- Status in this repo: kept as historical/reference code
- Notes:
	- Full-tree/syntax-receiver style is generally less efficient at scale.
	- The source file itself notes this approach as deprecated in favor of incremental generators.

### 2) `CsvIncrementalGenerator6` (incremental, .NET 6 style)
- Pattern: `IIncrementalGenerator` + `CreateSyntaxProvider`
- Attribute: `CsvIncrementalSerializableAttribute6`
- Output shape: generates static extension methods in `CsvIncrementalSerializer6`
- Purpose: shows early incremental pipeline approach before newer APIs.

### 3) `CsvIncrementalGenerator` (incremental, current primary)
- Pattern: `IIncrementalGenerator` + `ForAttributeWithMetadataName`
- Attribute: `CsvIncrementalSerializable`
- Output shape: generates `partial` members directly into each annotated class/record
- Purpose: recommended and modern approach in this solution.
- Behavior details:
	- Works with classes and records.
	- Skips non-public and static properties.
	- Emits per-type generated file names like `CsvIncrementalSerializer.<TypeName>.g.cs`.

## Prerequisites

- .NET SDK that supports the solution target frameworks (projects target modern .NET, including net10.0 in sample/test projects)
- Visual Studio / VS Code with C# tooling

## Build and run

From solution root:

```powershell
dotnet restore
dotnet build Generators.sln
dotnet run --project ConsoleApp1/ConsoleApp1.csproj
```

The console output should show CSV rows generated by compile-time emitted code.

## How to use the generator in a consumer project

1. Reference the generator project/package as an analyzer.
2. Mark your type with `[CsvIncrementalSerializable]`.
3. Make the type `partial` for the primary generator style used here.
4. Call generated APIs (`ToCsv`, `ToCsvHeader`, `TypeName.ToCsv(IEnumerable<TypeName>)`).

Example:

```csharp
using CsvIncrementalSerializer;

[CsvIncrementalSerializable]
public partial class Person
{
		public string Name { get; set; } = string.Empty;
		public int Age { get; set; }
}

var person = new Person { Name = "Jane", Age = 42 };
var row = person.ToCsv();
```

For project references, the pattern used in `ConsoleApp1.csproj` is:

```xml
<ProjectReference Include="..\Generators\Generators.csproj"
									OutputItemType="Analyzer"
									ReferenceOutputAssembly="false" />
```

## Tests overview

`GeneratorTests` contains two categories:

- Snapshot tests (`CsvIncrementalTest.cs`)
	- Validate generated source text.
	- Use Verify to compare output to approved snapshots in `GeneratorTests/Snapshots/`.
- Integration tests (`CsvIntegrationTests.cs`)
	- Compile and execute code against generated APIs.
	- Validate actual CSV runtime strings.

## SDK-supplied source generator tests

In addition to custom CSV generator tests, this solution includes unit tests for source generators that ship with the .NET SDK/libraries:

- JSON source generation (`GeneratorTests/JsonTest.cs`)
	- Compares traditional `JsonSerializer.Serialize(...)` with source-generated context serialization via `JsonSerializerContext`.
	- Demonstrates `[JsonSerializable]` and `[JsonSourceGenerationOptions]` usage.
- Logger message generator (`GeneratorTests/LoggerTest.cs`)
	- Uses `[LoggerMessage]` on a partial logging method and validates the generated logging path through a fake logger.
- Regex source generation (`GeneratorTests/RegexText.cs`)
	- Uses `[GeneratedRegex]` and compares generated regex behavior to a traditional runtime-compiled regex.

These tests are useful onboarding references for developers who want examples of both custom generators and SDK-provided generators in the same codebase.

Run tests:

```powershell
dotnet test GeneratorTests/GeneratorTests.csproj
```

## Verify snapshot workflow

When generator output intentionally changes:

1. Run tests.
2. Inspect generated `*.received.txt` files for failing snapshots.
3. If changes are correct, accept them by replacing corresponding `*.verified.txt` content.
4. Re-run tests to confirm all pass.

Notes:
- Snapshot files in this repo live in `GeneratorTests/Snapshots/`.
- Verify settings are configured in `CsvIncrementalTest` (`UseDirectory("Snapshots")`, scrubbers, and naming settings).

## Development notes

- `Generators` targets `netstandard2.0`, which is typical for Roslyn analyzer/source generator distribution.
- Test helpers in `GeneratorTests/Helpers/TestHelpers.cs` execute generators twice and assert incremental caching behavior.
- If you add a new generator stage or output behavior, update both integration tests and snapshot baselines.

## Suggested onboarding path

1. Read `Generators/CsvIncrementalGenerator.cs` to understand the recommended pattern.
2. Run `ConsoleApp1` to see generated methods used in normal code.
3. Run `GeneratorTests` and inspect snapshots to understand expected generated output shape.
4. Compare with `CsvIncrementalGenerator6.cs` and `CsvGenerator.cs` to learn API evolution.
